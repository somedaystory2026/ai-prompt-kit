# AI 이미지·영상 자동 생성+저장 4단계 시스템

> **추천 AI**: Claude 또는 ChatGPT  
> 💡 대본 → 캐릭터 참조 이미지 → 핵심 장면 이미지 → Grok 영상 프롬프트까지 4단계로 자동화합니다.  
> 이미지 생성 도구: **Whisk** / 영상 생성 도구: **Grok Imagine**

## 사용 흐름

```
STEP 1 대본 작성 (Claude/GPT)
  ↓
STEP 2 캐릭터 참조 이미지 프롬프트 생성 (Claude/GPT) → Whisk에 입력
  ↓
STEP 3 핵심 장면 이미지 프롬프트 N장 생성 (Claude/GPT) → Whisk에 입력
  ↓
STEP 4 영상 행동 묘사 프롬프트 생성 (Claude/GPT) → Grok Imagine에 입력
```

---

## STEP 1 — 스토리텔링 대본 작성

```
[사용자 입력]
- 주제: <주제를 입력하세요>
- 목표 분량: <N분 내외>

주어진 주제를 바탕으로,
오디오북과 영상 제작에 바로 사용할 수 있는
몰입감 있는 스토리텔링 대본을 작성해줘.

작성 기준:
- 주제의 성격에 맞는 어조를 스스로 판단해 선택할 것
- 첫 문장은 청자의 관심을 끌 수 있는 강력한 도입으로 시작할 것
- 전체 이야기는 다음 구조를 따를 것:
  발단 → 전개 → 위기 → 절정 → 결말
- 실제 나레이션으로 읽었을 때
  호흡이 자연스럽고 장면이 떠오르도록 작성할 것
- 메시지가 있다면 설교처럼 보이지 않게 이야기 흐름 속에 자연스럽게 녹일 것

출력 형식:

[전체 이야기 요약]
- 전체 흐름을 파악할 수 있는 3줄 요약

[최종 나레이션 대본]
- 장면 구분 없이 자연스럽게 이어지는 낭독용 대본
```

---

## STEP 2 — 등장인물 추출 & 캐릭터 참조 이미지 프롬프트

```
[사용자 입력]
- 원하는 이미지 스타일: <선택 또는 직접 입력>

※ 선택 예시
- photorealistic (사진처럼 사실적인 느낌)
- 시네마틱 3D 애니메이션 (실제 영화처럼 웅장하고 화려한 영상미)
- 픽사풍 3D 캐릭터 (귀엽게 과장된 비율과 사실적인 소품 질감)
- 3D 디즈니 스타일 (크고 맑은 눈망울과 부드럽고 매끈한 피부 표현)
- 세미 리얼리스틱 3D 일러스트 (캐릭터의 특징을 살리면서도 실사처럼 정교한 묘사)
- 동화책 느낌의 스타일라이즈드 일러스트 (포근한 색감과 정감 있는 손그림 질감)
- 다크 판타지 일러스트 (어둡고 무거운 분위기의 웅장한 중세 판타지 느낌)

위에서 생성한 스토리텔링 대본을 기준으로,
이야기에 등장하는 주요 인물을 추출하고
각 인물별 캐릭터 참조 이미지용 프롬프트를 만들어줘.

규칙:
- 이야기 흐름상 중요한 인물만 선별할 것
- 각 인물은 고유한 캐릭터로 인식될 수 있도록 작성할 것
- 모든 캐릭터는 전신(full body)이 명확히 보이는 참조 이미지 기준으로 작성할 것
- 외형 설명은 이미지 생성에 필요한 핵심 특징만 포함할 것
- 얼굴 클로즈업이나 상반신 위주의 묘사는 피하고,
  자세·실루엣·복장 전체가 드러나도록 설정할 것
- 성격/분위기/이야기 속 역할이 자연스럽게 드러나도록 할 것
- 모든 캐릭터는 동일한 이미지 스타일을 적용할 것
- 이 단계에서 생성된 이미지는 이후 단계에서 reference image로 반복 사용됨을
  고려해, 중립적인 포즈와 안정적인 구도를 유지할 것

출력 형식:

[Character List]
- 등장인물 이름(또는 역할명) + 역할 한 줄 요약

[EN – Character Reference Image Prompts]
- 영어로만 작성
- 한 줄 = 한 명의 캐릭터 참조 이미지 프롬프트
- 모든 프롬프트에 full body, full figure, clear silhouette 개념이 자연스럽게 포함될 것
- 캐릭터 수만큼 생성

[KR – Character Description]
- 번호로 구분 (예: 1번 캐릭터, 2번 캐릭터)
- 각 설명 첫 줄에 다음 문장 포함:
  [이 캐릭터의 참조 이미지입니다]
- 전신 기준으로, 이후 모든 장면에서 동일 인물로 인식될 수 있도록 설명할 것
- 위 영어 프롬프트와 1:1 대응되게 작성

단, 이미지 생성 도구는 쓰지 말고 텍스트로만 답해줘
```

## STEP 3 — 핵심 장면 이미지 프롬프트 생성 (N장)

```
[사용자 입력]
- 생성할 이미지 총 개수 (N): <N장>

스토리 대본을 기준으로,
Whisk에서 바로 사용할 수 있는 이미지 프롬프트를 생성해줘.

규칙:
- N은 전체 대본 기준 이미지 총 개수
- 시각적으로 가장 중요한 핵심 장면만 선별할 것
- 장면은 반드시 시간 순서대로 배열할 것
- 한 이미지에는 하나의 핵심 상황만 담을 것
- 캐릭터 외형은 직접 설명하지 말고
  "based on reference images"라고만 표현할 것

출력 형식:

[EN – Image Prompts]
- 영어로만 작성
- 한 줄 = 하나의 이미지 프롬프트
- 각 프롬프트 사이에 한 줄 공백
- 그대로 복사해 사용할 수 있어야 함

[KR – Prompt Meaning]
- 번호로 장면 구분 (예: 1번 장면, 2번 장면…)
- 각 장면 첫 줄에 포함:
  [참조 이미지 선택: 인물1, 인물2...]
- 위 영어 프롬프트와 1:1 대응

단, 이미지 생성 도구는 쓰지 말고 텍스트로만 답해줘
```

---

## STEP 4 — 영상 행동 묘사 프롬프트 (Grok 영상용)

```
이미지 프롬프트에 해당하는 장면을 기준으로,
각 장면을 짧은 영상으로 만들기 위한
영상 행동 묘사 프롬프트를 작성해줘.

규칙:
- 한국어로만 작성
- 장면의 시간 흐름 순서를 그대로 유지할 것
- 외형, 의상, 배경, 스타일은 다시 설명하지 말 것
- 카메라, 렌즈, 조명, 샷 구성 같은 기술적 표현 금지
- 한 줄 = 한 장면
- 한 장면에는 하나의 핵심 행동/변화만 담을 것
- 정적인 장면이 자연스럽게 살아 움직이도록 행동만 묘사할 것

출력 형식:
[영상 프롬프트]
- 한 줄 = 한 장면
- 각 줄 사이에 한 줄 공백
- Grok에서 그대로 복사해 사용할 수 있어야 함
```

## 팁

- STEP 2에서 생성한 캐릭터 참조 이미지는 Whisk에 업로드한 뒤 STEP 3 프롬프트와 함께 사용
- 동일 캐릭터 일관성 유지: Whisk의 reference image 기능 활용
- STEP 4 영상 프롬프트는 Grok Imagine → 이미지 선택 후 영상 변환 시 입력
- 영상 생성 자동화: AI연구소 Studio 확장프로그램의 오토그록 기능 연동 가능
