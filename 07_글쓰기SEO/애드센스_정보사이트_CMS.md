# 애드센스 승인용 정보 사이트 + 관리자 모드 (CMS-lite) 생성기

> **추천 AI**: Claude (Claude Code 사용 시 더 효과적)  
> 💡 Replit에서 바로 실행 가능한 완성형 정적 웹사이트 + 워드프레스 느낌의 관리자 모드를 생성합니다.

## 프롬프트

```
너는 숙련된 웹사이트 디자이너이자 프론트엔드 개발자이며,
SEO 친화적인 정보 사이트를 만드는 전문가다.

[입력 변수] — 아래 항목을 채워서 프롬프트와 함께 사용하세요

- 사이트명: {{SITE_NAME}}
- 사이트 한줄 소개: {{SITE_TAGLINE}}
- 사이트 주제: {{SITE_TOPIC}}
- 메인 타깃 독자: {{TARGET_AUDIENCE}}
- 톤앤매너: 신뢰감 있고 친절하며 과장 없는 정보형
- 메인 컬러: {{MAIN_COLOR}}
- 서브 컬러: {{SUB_COLOR}}
- 운영자명: {{OWNER_NAME}}
- 운영자 소개 한줄: {{OWNER_BIO}}
- 연락 이메일: {{CONTACT_EMAIL}}
- 기본 도메인 표기: {{SITE_URL}}

[핵심 목표]
단순 블로그처럼 보이면 안 되고, 실제 브랜드형 정보 사이트처럼 보여야 한다.
구글이 중요하게 보는 사람 중심 콘텐츠, 신뢰 요소, 사이트 완성도, 명확한 주제 구조를 갖춘 사이트를 만들어라.

[중요 원칙]
- YMYL 주제(건강, 금융, 투자, 대출, 세금, 법률) 금지
- 가짜 후기, 가짜 수치, 가짜 자격, 가짜 회사 정보, 허위 경험담 절대 금지
- lorem ipsum 더미 텍스트 금지. 실제 사람이 읽을 수 있는 자연스러운 한국어 문장으로 작성
- 모바일 반응형, 빠른 로딩, 깔끔한 UI, 명확한 카테고리 구조, 신뢰 페이지 포함
- 광고 코드, 허위 회사 주소, 허위 전화번호 금지
- 연락 수단은 이메일 기반으로만 처리
- 결과물은 설명이 아니라 "실행 가능한 전체 프로젝트 코드" 중심으로 제공

[기술 방향]
- Replit에서 바로 실행 가능한 프로젝트
- 정적 사이트 중심 (HTML/CSS/JS)
- 별도 백엔드 없이도 완전 동작
- 브랜드명, 이메일, 색상, 소개 문구, 카테고리, 글 데이터를 한군데에서 쉽게 수정 가능
- 콘텐츠는 data 파일(json/js 배열)로 분리
- sitemap.xml, robots.txt, favicon, 404 페이지 포함
- Article / Breadcrumb / FAQ 구조화 데이터(JSON-LD) 포함

[사이트 구조 — 반드시 포함할 페이지]
1. 홈 / 2. 카테고리 목록 / 3. 카테고리별 상세 / 4. 글 상세 (최소 15개)
5. 사이트 소개(About) / 6. 운영자 소개(Author) / 7. 문의하기(Contact)
8. 개인정보처리방침 / 9. 이용약관 / 10. 면책고지
11. 사이트맵 / 12. 404 페이지 / 13. 칼럼 목록 / 14. 칼럼 상세 (최소 3개)
15. 관리자 모드(admin)

[워드프레스 느낌의 관리자 모드 — /admin 경로]
- 로그인 전/후 화면 구분 (localStorage 기반 세션, 실제 보안 아님을 명확히 고지)
- 좌측 사이드바 + 우측 메인 편집 패널 구조
- 좌측 메뉴: 대시보드 / 일반 글 관리 / 칼럼 관리 / 카테고리 / 사이트 설정
- 대시보드: 총 글 수, 총 칼럼 수, 카테고리 수, 최근 수정 콘텐츠
- 글 관리: 글 목록 / 새 글 작성 / 수정 / 삭제 UI
  (제목, 슬러그, 요약, 카테고리, 본문, FAQ, 관련 글, 추천글 여부, 발행 상태, 작성일/수정일)
- 칼럼 관리: 운영자 칼럼 전용 목록 / 새 칼럼 작성 / 수정 / 삭제
- 사이트 설정: 사이트명 / 한줄 소개 / 운영자명 / 이메일 / 메인 컬러 / 서브 컬러
- 데이터 관리: localStorage 기반 저장 / JSON export / JSON import
- 실제 데이터베이스/인증/권한 관리가 없음을 명시
  "추후 Supabase, Firebase, Git 기반 CMS로 확장 가능" 안내 포함

[운영자 이름 클릭 흐름]
- 사이트 전반의 운영자명 클릭 → /author/ 이동 (운영자 소개 + 칼럼 목록)
- 관리자 세션이 없으면: "운영자가 정리한 칼럼을 읽어보세요"
- 관리자 세션이 있으면: "새 칼럼 작성하기" 버튼 노출 → /admin/의 칼럼 작성 화면으로 이동

[홈페이지 필수 요소]
히어로 섹션 / 대표 카테고리 4~6개 / 최신 글 섹션 / 추천 글 섹션
운영 목적 소개 / 편집 원칙 요약 / 운영자 소개 미니 박스 / 칼럼 섹션 미리보기
문의하기 CTA / 푸터에 개인정보처리방침·이용약관·문의하기·운영자 소개 링크

[콘텐츠 요구사항]
- 글 상세 페이지 최소 15개, 운영자 칼럼 최소 3개
- 각 글에 반드시 포함: 제목 / 요약 문장 / 작성자 정보 / 작성일·수정일 / 목차 / 본문
  핵심 요약 박스 / 초보자가 자주 실수하는 포인트 / 체크리스트 / 관련 글 링크
- 글 구조: 문제 제기 → 핵심 설명 → 실제 팁 → 주의사항 → 정리
- 날짜는 최근 몇 달 안에서 자연스럽게 분산 배치 (동일 날짜 몰아넣기 금지)
- "완벽 정리", "무조건", "100% 승인", "충격" 같은 표현 자제

[폴더 구조]
/index.html
/about/, /author/, /contact/, /categories/, /posts/, /columns/, /admin/
/privacy/, /terms/, /disclaimer/, /sitemap/, /404.html
/robots.txt, /sitemap.xml
/assets/css/, /assets/js/, /assets/icons/
/data/site.config.js, /data/categories.js, /data/posts.js, /data/columns.js

[출력 방식]
1. 폴더 구조 설계
2. 공통 레이아웃과 스타일 시스템
3. 홈 / 카테고리 / 글 상세 / 칼럼 / 신뢰 페이지 / 관리자 페이지
4. 콘텐츠 자연스러운 한국어로 채우기
5. SEO 메타와 구조화 데이터 반영
6. Replit에서 바로 미리보기 가능한 상태로 완성
7. README 포함 (실행 방법 + 수정 방법)

이제 위 요구사항을 모두 반영해서,
{{SITE_TOPIC}} 주제에 맞는 완성형 한국어 정보 사이트를 제작해라.
설명보다 실제 프로젝트 코드 중심으로 출력하라.
```

## 입력 변수 예시

```
- 사이트명: 생활정보마당
- 사이트 한줄 소개: 일상에서 바로 쓰는 생활 정보 모음
- 사이트 주제: 생활정보 / 일상 꿀팁
- 메인 타깃 독자: 20~40대 생활 정보 검색 초보자
- 메인 컬러: #6c5ce7
- 서브 컬러: #a29bfe
- 운영자명: 홍길동
- 운영자 소개 한줄: 생활 속 유용한 정보를 직접 써보고 정리합니다
- 연락 이메일: hello@example.com
- 기본 도메인 표기: lifetips.vercel.app
```

## 팁

- Claude Code에서 실행 시 여러 파일로 분할 생성 가능 (더 안정적)
